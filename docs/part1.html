First step: downloading Python

Your Mac actually comes with Python built-in, but don’t use it! This is referred to as the system Python and you probably don’t want to mess with it as it may be used internally by the operating system. I like to use the miniconda Python distribution: https://docs.conda.io/en/latest/miniconda.html

Or if you have the newer M1 Macbook (pro/air), I recommend using the miniforge distribution:
https://github.com/conda-forge/miniforge

Go to the link above and follow the installation instructions.

On a Mac, your Python will typically be installed under
/Users/<your_user_name>/miniconda3 if you’re using miniconda, or
/Users/<your_user_name>/miniforge3 if you’re using miniforge.

What is nice about having the new Python install is that it’s completely isolated from your system, if you want to start all over again, just simply delete the folder above and re-install a fresh Python.

At the moment (August 12th, 2022), I recommend working with at least Python versoin 3.8. There are newer Python versions out there, but for the most part, version 3.8 is chosen because many of the libraries we’ll be using will depend on the this older stable version.

For reference, once you’ve installed conda, you can refer to their documentation to learn how to use it (https://docs.conda.io/projects/conda/en/latest/user-guide/getting-started.html). In this document, I go over some commands briefly to get quickly get started. But it’s essential that you know how to do some basic things with conda

Terminal crash course

It’s essential that you know your way around the “command line” or Terminal in Mac/Linux. These functions are routinely used:

  • ls list the contents of current folder
  • pwd prints out the current directory that you are in now
  • cd change directory
    • cd ../ go up a ditectory
    • cd my_folder go into a directory named my_folder
  • mv move or rename a file. mv <source> <target>. Example:
    • mv img_01_ch02.tif img_z1_GFPchannel.tif renames img_01_ch02.tif to img_z1_GFPchannel.tif
  • cp copy a file. cp <source> <target>. Example:
    • cp file1.dat ../file1_copy.dat copy file1.dat in the current folder as another file called file1_copy.dat into one level of directory up from the current one.

Python Environments

Before you do install any packages in your new Python installation, it’s always good to create a new ‘environment’ to work in. Environments are a way to create a sandbox for a specific Python projects. So if different projects will require different sets of packages that depend on different versions of Python or its required packages, you won’t have to worry about running into package incompatibilities! The Python package ecosystem is huge!! and sooner or later you’ll run into this problem as you install more and more packages. Once you install Python as outlined above (e.g. using (mini)conda or miniforge), you’ll automatically have an environment called base. I recommend NOT installing packages into your base environment. Instead, create a new one by running the command:

conda create -n my_new_project python=3.8

This will create a new environment called my_new_project, you should choose whatever name is appropriate or to your liking. (e.g. image_analysis). After creating this environment, you can now activate it to go into your Python ‘sandbox’ by running:

conda activate my_new_project

Now that you’re in the environment, you can begin installing useful Python packages that are commonly used for a nice and friendly Python experience:

  • jupyter, for interactive Python ‘notebooks’
  • matplotlib, for plotting data and making figures
  • numpy, for doing any numerical computation

To exit the current environment, you can run the command:
conda deactivate

Or simply run conda activate another_environment to deactivate the current one and activate another_environment.

To see all of the environments that you currently have you can run:
conda env list

Installing packages

To install a new Python package, you can either use conda or pip. But keep in mind that they’re not necessarily compatible with each other! So keeping that in mind, you should try to stick with just one way to install packages (e.g. try using conda or pip for ALL installs). There are cases where you can (or need to use both because not all packages are available in conda or pip alone).

Conda packages are kept in an online repository, the default one is conda-forge
To install using conda:
conda install jupyter

To install using conda with a specific “channel”. A comprehensive and popular one is “conda-forge” (https://conda-forge.org). To specify that you want a package from a specific channel, you pass the extra -c <channel_name> argument to the conda command:
conda install jupyter -c conda-forge
